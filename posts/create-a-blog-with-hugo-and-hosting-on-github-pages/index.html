Create a Blog With Hugo and Hosting on GitHub Pages

Jelen bejegyzésben megmutatom, hogyan hoztam létre ezt a blog oldalt.

Áttekintés#

Az oldalt a Hugo nevű statikus oldal generálóval készítem, publikusan elérhetővé pedig a GitHub Pages segítségével teszem.

GitHub-on két repository-t készítettem, az egyik tartalmazza az oldal forráskódját és a tartalmakat, a másik pedig a generált oldalt.

Blog létrehozása Hugo-val#

Hugo telepítése#

A telepítés igen egyszerű, a hivatalos útmutató alapján bármilyen operációs rendszerre fel tudjuk telepíteni.

Blog oldal létrehozása#

A blog oldal létrehozása a hugo programmal:

hugo new site blog

amely az adott könyvtárban kiadva létrehoz egy blog könyvtárat. Ezen könyvtárat inicializáltam mint git repository:

cd blog
git init

Theme választása#

Az oldal létrehozás után egy theme-át kell választani és beállítani. A Hugo Themes oldalon nagy választék van, innen a PaperMod theme-át választottam.

Theme beállítása#

A repository-hoz hozzáadtam submodule-ként a PaperMod theme-át:

git submodule add \
    https://github.com/adityatelange/hugo-PaperMod.git \
    themes/PaperMod \
    --depth=1

vagy

git submodule--helper add \
    https://github.com/adityatelange/hugo-PaperMod.git \
    themes/PaperMod \
    --depth=1

Majd a config.toml fájlba megadtam, hogy ez legyen az oldal által használt theme:

echo theme = \"PaperMod\" >> config.toml

A beállítások eredményességéről meggyőződhetünk a hugo server indításával:

hugo server -D

A -D kapcsoló jelentése, hogy draft módban indítjuk.

Az oldalt a http://localhost:1313/blog-on meg tudjuk tekinteni.

Blog Page létrehozása GitHub-on#

Blog forráskód repository#

GitHub-on létrehoztam egy blog-hugo nevű repository-t és beállítottam mint remote repository.

Létrehoztam egy .gitignore fájlt az alábbi tartalommal:

.hugo_build.lock

# Generated files by hugo
/public/
/resources/_gen/

Majd feltettem github-ra:

git add .
git commit -m "init blog"
git push -u origin master

Blog statikus oldal repository#

A Hugo által generált oldalt egy külön repository-ban tárolom.

A Hugo a generált oldalt a public könyvtárba hozza létre és ezen repository ezt fogja tartalmazni.

GitHub-on létrehoztam egy blog nevű repository-t és submodule-ként felvettem a public könyvtárhoz:

git submodule--helper add git@github.com:<username>/blog.git public

A fő repository-ban változásként látni fogjuk a .gitmodules-t és a public könyvtárat. Adjuk hozzá a verziókövetőhöz:

git add .
git commit -m "add public submodule"
git push

GitHub Pages beállítása#

A GitHub-on a blog repository Settings-ei közt a Pages menüben Source-ként válasszuk ki a brach-ünket és folder-ként a / (root)-ot.

Ezt követően a fő repository-ban levő Hugo config fájlt módosítanunk kell, hogy a GitHub Page-ünk url-jét tartalmazza.

A GitHub Page-ünk url-je két részből tevődik össze:

  • az első rész: a github user nevünk, amit a .github.io követ
  • a második rész: a repository neve, ami jelen esetben blog

Ezen felül az oldalunk címét is meg kell adni, lecserélve a generált címet.

Ezen beállításokat a config.toml fájlban tehetjük meg:

baseURL = 'https://<user-name>.github.io/blog/'
languageCode = 'en-us'
title = 'Blog'
theme = "PaperMod"

Ezzel előkészítettük a terepet az oldal generáláshoz és publikáláshoz.

Módosításainkat töltsük fel:

git add .
git commit -m "update config"
git push

Hugo statikus oldal generálás#

A blog oldal legenerálása:

hugo

Ezt követően a public könyvtárba lépve a generált oldalt fel tudjuk tölteni:

cd public
git add .
git commit -m "init blog"
git push

A feltöltést követően nemsokára a GitHub Page deploy lefut és a Blog oldal elérhetővé válik.
